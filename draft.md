Update "data.yml" with the information currently in "index.html". Do not run the onsave utility yet as it will overwrite the index.html. Once completed, proceed to copy index.html as reference_index.html, then run the the onsave command to generate a new index.html and ensure it matches reference_index.html. The onsave util is going to keep running in watch mode so it will need to run in the background will a frivilous change is made to data.yml and then un-done just so that it updates the file (please don't hang). The onsave command is my creation, it's in ~/go/bin/onsave, and it does not have any subcommands like "help" - if you do that it will also hang in watch mode.
